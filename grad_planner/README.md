# DGIST Graduation Planner

DGIST 융복합대학 기초학부의 「교육과정표(2020학번 이후, 2025.11.17.)」를 기반으로 만든 순수 정적 졸업 이수 시뮬레이터입니다. 별도 빌드·서버·데이터베이스 없이 GitHub Pages에 바로 배포할 수 있습니다.

## 주요 기능

- 각 학년을 `1학기 → 2학기 / 여름 → 겨울` 순서의 2×2 Z형 구조로 구성하고 과목을 드래그하거나 버튼으로 배정
- 기본 4학년에 더해 5·6학년 초과학기를 필요할 때 순차 생성
- 정규 1·2학기에서 엇학기를 시작·해제하면 그 시점부터 이후 정규학기의 봄·가을이 누적 반전(계절학기는 영향 없음)
- 표준 개설학기와 다른 학기에 넣을 때 확인 후 허용
- 2020 / 2021~2024 / 2025 이후 학번별 기초·심화·총학점 판정
- `2과목 중 1과목` 같은 선택군을 개별 필수가 아닌 이수 블록으로 표시·검증
- 269개 전체 교과목을 `기초필수·기초선택·심화필수·심화선택` 네 풀에 모두 펼쳐 표시하고, 각 풀 안에서 교양·기초과학·기초공학·비트랙/융합·연구·인턴십 및 전공 트랙별로 구분
- 전공 트랙과 과목 수가 많은 인문사회 교양은 접기/펼치기를 지원하고 영어·한국어·예체능 교양은 바로 표시
- 공학선택을 컴퓨터공학과 분리하고 창의기계설계, 회로이론 이론·실습 세트, 화학공학개론을 하나의 선택 블록으로 표시
- 왼쪽 과목 선택 패널을 독립 스크롤 영역으로 고정해 오른쪽 이수계획을 내리지 않고도 전체 과목 탐색
- 모든 교과목 카드와 배정 카드에 개별 필수·필수 선택군·전공/부전공 지정·트랙 선택·인정 상한 역할 배지 표시
- 기초선택은 기초필수 학점에 포함하지 않고, 심화선택은 트랙·비트랙/융합·UGRP·인턴십 필수영역을 대신하지 않도록 분리 판정
- 졸업 자체와 전공·복수전공·부전공 표기요건을 분리하고 트랙 간 중복 인정 최대 6학점 판정
- 비트랙/융합, UGRP, 인턴십, 외국인 학생 한국어 요건 확인
- 직접 입력 시 네 가지 필수·선택 구분을 먼저 선택하고, 해당 구분에 맞는 인정영역과 대체 가능 지정과목만 표시
- 계획 데이터 로컬 저장 및 JSON 내보내기/불러오기
- A4 가로형 졸업 진단 리포트와 학기별 계획표 PDF/인쇄
- 반응형 UI, 키보드 포커스, 라이트/다크 테마

> 이 도구는 계획 보조용입니다. 공식 졸업사정은 학칙, 교육과정 운영 및 이수에 관한 요령, 학사팀 확인을 우선합니다.

## 실행

루트 디렉터리에서 정적 서버를 실행합니다.

```bash
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000`을 엽니다. 외부 패키지는 필요하지 않습니다.

판정 엔진 테스트:

```bash
node tests/engine.test.js
```

브라우저 상호작용과 PDF 회귀 테스트는 정적 서버를 켠 상태에서 별도 터미널로 Chrome 디버깅 인스턴스를 실행한 뒤 수행합니다.

```bash
google-chrome --headless=new --no-sandbox --remote-debugging-port=9222 \
  --user-data-dir=/tmp/dgist-grad-test http://127.0.0.1:8000/
node tests/browser-smoke.mjs
```

이 테스트는 전체 269과목 표시, 네 가지 필수·선택 판정, 직접 입력 양식, 정규·계절학기 드래그, 누적 엇학기, 5·6학년 초과학기, 졸업 진단과 A4 PDF 생성을 검증합니다.

## GitHub Pages 배포

1. 이 디렉터리의 파일을 GitHub Pages 게시 원본 아래 `grad_planner/` 디렉터리에 올립니다.
2. 저장소 `Settings → Pages`에서 `Deploy from a branch`를 선택합니다.
3. 기본 브랜치의 `/ (root)`를 게시 원본으로 지정합니다.

배포 주소는 `/grad_planner/`이며 별도 빌드 단계가 필요 없습니다.

## 유지보수 구조

- `assets/curriculum-data.js`: 과목, 개설학기, 필수·선택 속성, 이수 블록, 학점, 트랙, 학번·트랙별 규칙
- `assets/graduation-engine.js`: 데이터에 따라 학점과 지정필수 요건을 판정하는 순수 함수
- `assets/app.js`: 화면, 로컬 저장, 가져오기/내보내기, 인쇄, 학기 경고
- `assets/app.css`: ICML26 페이지의 카드·필터·요약 스트립 시각 언어를 따른 반응형/인쇄 스타일
- `tests/engine.test.js`: 학번 기준, 지정필수, 대체 인정, 한국어, 인턴십 인정학점 회귀 테스트

새 과목은 `courses`에 다음 형식으로 추가합니다.

```js
add(
  "stable_course_id",
  "교과목명",
  3,
  "advanced",            // basic | advanced
  "track",               // categories에 정의된 이수 구분
  3,                       // 표준 학년
  ["spring"],              // spring | summer | fall | winter
  { tracks: ["computer"], requirement: "required" } // required | elective
);
```

트랙 지정필수는 같은 파일의 `trackRules`, 학번별 학점은 `cohortRules`에서 관리합니다. 데이터 변경 시 원본 교육과정표와 회귀 테스트 결과를 함께 검증합니다.

## 반영한 핵심 원문 기준

- 2020학번: 기초 62, 심화 72, 총 134학점
- 2021학번 이후: 기초 58, 심화 72, 총 130학점
- 2025학번 이후: 기초 컴퓨터공학 6학점, 인문사회 18학점
- 수학 9학점: 공학수학Ⅰ·다변수 미적분학 + 지정 선택군 1과목
- 물리·화학·생명과학 합계 18학점과 각 영역 지정 이론·실험
- 공학선택 3, 영어 4, 비트랙/융합 6, UGRP 6, 트랙 최소 27학점
- 2020학번 예체능 4학점
- 2023학번 이후 인턴십 필수 1학점(이전 2학점)
- 2025학번 이후 외국인 학생 한국어Ⅰ·Ⅱ 중 1과목 또는 공식 면제

선수과목, 폐지·대체교과목의 개인별 적용, 재수강, 학위연계과정 등 행정 예외는 자동으로 확정하지 않으며 화면에 그 한계를 명시합니다.
